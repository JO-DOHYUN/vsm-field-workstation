# 마이그레이션 순서

## A. 먼저 고정할 것
- 기존 프로젝트는 **백업만** 하고 건드리지 않는다.
- 이 새 프로젝트에서만 구조 전환 작업을 한다.

## B. 1차 이관
- `FrameRec`, `StatsRec` 의미 보존
- 20바이트 패킷 포맷 보존
- 시리얼 속도 921600 보존
- 로그 `.bin` 포맷 보존

## C. 2차 이관
- 기존 룰 로드(JSON) -> `rule_engine` 모듈로 분리
- 기존 주기/TTL 계산 -> `timing_evaluator` 분리
- 기존 값 해석 -> `signal_decode` 분리

## D. 3차 이관
- QML `TimingPage`와 `ValuePage`를 실제 데이터 모델에 연결
- 경보 탭 모델 연결
- 세션/재생/탐색 UI 완성

## E. 마지막
- 차량/도로 애니메이션 탭 추가
- 패키징/설치형 배포 정리
