# Release Checklist

## Current RC Status
- Windows VSM portable RC is the field artifact.
- Latest generated RC: `out/build/x64-Release/portable_field_rc_20260604`.
- Code/function RC closure requires Release build, full `ctest` 22/22, exe startup smoke, portable deploy smoke, and CSM PlatformIO build.
- Fresh HIL/vehicle stability and vehicle control success are field-only gates, not automatic program-smoke claims.

## Build and Test
- shared preset Debug configure/build passes
- shared preset Release configure/build passes
- Release `ctest` passes 22/22
- targeted QML/state probes cover graph selection stability, visible DLC, replay diagnostics, ControlPage evidence authority, and Live transport diagnostics

## Packaging
- portable deploy folder generated by `deploy_release.bat`
- portable folder contains exe, Qt runtime, data, helper scripts, `FIELD_VALIDATION_KO.md`, packaging notices, and `RELEASE_MANIFEST.json`
- WiX MSI stage is deferred unless install/uninstall is explicitly validated

## Runtime Smoke
- app starts and reaches main window
- live connect/disconnect
- logging start/stop/save/discard
- replay load/play/pause/seek
- issue navigation
- timing/value/alarm coherence
- recent-window graph and full-range overview graph
- portable exe starts from the regenerated RC folder

## Release Metadata
- build metadata matches intended baseline
- `RELEASE_MANIFEST.json` contains exe hash, model hash, package id, protocol version, known issues, and unverified HIL/field items
- session log is emitted
- SBOM placeholder or generated SBOM is present
- third-party notices are present
- known issues captured

## Deferred / Field-Only
- vehicle bus0/bus1 hot-plug stability
- vehicle control TX success, confirmed only by matching `CAN_TX_RAW`
- Android USB-OTG or wireless CSM transport feasibility
- code signing
